Restore Backup

Upload a previously exported backup ZIP to restore all database tables.

Danger Zone

Restoring a backup will replace all existing data in every table. This action cannot be undone. Export a fresh backup first if you need to preserve the current state.

About Backups

  • Format — Backup is a ZIP file containing a CSV for each table and a manifest.json
  • Security — Backups contain password hashes (bcrypt) and Fernet-encrypted credentials. Store backups securely.
  • Sessions — User sessions are not included in backups. Users will need to log in again after a restore.
  • Size limit — Restore files must be under 100 MB uncompressed.
  • Audit trail — Both export and restore operations are recorded in the audit log.
